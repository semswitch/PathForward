# Skill-Adjacency & Reskilling Notes (Synthetic)

> Synthetic demonstration content for PathForward. Obviously fictional.

The reskilling problem is rarely a *content* gap — it is an *engagement / capacity* gap.
Displaced workers often hold **transferable** skills adjacent to in-demand roles; the system's
job is to map those adjacencies, then plan around real capacity and accessibility needs.

## Example adjacencies
- **Data Center Technician → Cloud Engineer:** existing Networking (S12) and App Service (S05)
  exposure transfers; the gap is typically API Development (S01), Azure Functions (S02), and
  Infrastructure as Code (S08).
- **Manual QA Tester → DevOps Engineer:** test discipline transfers; gap is usually CI/CD (S06),
  GitHub Actions (S07), Containers (S09).
- **On-prem Sysadmin → Cloud Engineer:** strong infra base; gap centres on cloud-native services.

## Capacity guidance
- Workers with > 20 meeting hours/week show lower study completion.
- Optimal completion appears with 12–18 meeting hours and ≥ 15 focus hours.
- Schedule study blocks inside focus-heavy windows; adapt reminders to the worker's rhythm.

## Accessibility guidance
- Offer audio-narrated plans and an Oral Viva path for low-vision / screen-reader users.
- Provide live captions + a transcript for Deaf / hard-of-hearing learners.
- Allow adjustable pacing for neurodivergent learners; never autoplay audio without a stop control.
